# 模板方法模式 (Template Method)

## 意图

在父类中定义算法的骨架，将某些步骤延迟到子类中实现，使子类可以在不改变算法整体结构的情况下重定义某些特定步骤。

## 适用场景

- 多个类包含相同的算法骨架，仅部分细节步骤有差异，需要消除重复代码
- 希望控制子类可以扩展的部分（钩子），防止子类改变整体算法结构
- 需要复用公共行为同时允许各子类差异化实现特定步骤
- 构建框架时，框架定义算法流程，用户通过继承填充具体步骤

## 核心结构

```
  ┌──────────────────────────────────────────────┐
  │           AbstractClass (抽象类)              │
  │                                              │
  │  + templateMethod() [final，定义骨架]         │
  │      step1()   // 固定步骤，直接实现           │
  │      step2()   // 抽象步骤，子类必须实现       │
  │      hook()    // 钩子步骤，子类可选覆盖       │
  │      step3()   // 固定步骤，直接实现           │
  │                                              │
  │  # step2(): abstract                         │
  │  # hook(): 默认空实现                         │
  └──────────────────────────────────────────────┘
                       ▲
           ┌───────────┴───────────┐
           │                       │
  ┌──────────────────┐   ┌──────────────────┐
  │  ConcreteClassA  │   │  ConcreteClassB  │
  │  # step2()       │   │  # step2()       │
  │    // 实现A的逻辑 │   │    // 实现B的逻辑 │
  │  # hook()        │   │    // 未覆盖hook  │
  └──────────────────┘   └──────────────────┘
```

## 优点

- **消除重复代码**：公共算法骨架只在父类中定义一次
- **允许客户化扩展**：子类通过覆盖抽象方法和钩子定制行为，无需了解整体流程
- **控制扩展点**：父类决定哪些步骤可以被覆盖（钩子），哪些步骤是固定的

## 缺点

- **步骤越多维护越困难**：模板方法中步骤越多，子类继承负担越重
- **可能违反里氏替换原则**：子类覆盖步骤时若改变了语义，会导致行为不一致
- **继承不如组合灵活**：通过继承扩展不如策略模式（组合）灵活，子类与父类紧密绑定
