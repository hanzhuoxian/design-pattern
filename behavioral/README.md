# 行为型模式 (Behavioral Patterns)

行为型模式关注对象之间的**通信与职责分配**。它们描述了对象和类如何协作、如何划分职责，以及算法如何在对象之间流转，使系统更灵活、更易于扩展。

## 核心思想

- 将算法、行为与使用它们的对象解耦
- 明确对象间的通信方式，降低耦合度
- 在运行时动态改变对象的行为或职责

## 模式一览

| 模式 | 意图 | 核心关键词 |
|------|------|-----------|
| [责任链模式](./chain_of_responsibility/README.md) | 将请求沿处理者链传递，直到某个处理者处理它 | 链式处理、解耦发送者与接收者 |
| [命令模式](./command/README.md) | 将请求封装为对象，支持撤销、队列、日志等操作 | 封装请求、可撤销 |
| [解释器模式](./interpreter/README.md) | 为语言定义文法，并提供解释器处理该文法 | 文法、语法树 |
| [迭代器模式](./iterator/README.md) | 提供顺序访问聚合对象元素的方式，而不暴露其内部结构 | 遍历、封装迭代逻辑 |
| [中介者模式](./mediator/README.md) | 用一个中介对象封装对象间的交互，降低对象间的直接依赖 | 解耦、集中控制 |
| [备忘录模式](./memento/README.md) | 在不暴露内部状态的情况下，捕获并恢复对象的状态 | 快照、撤销恢复 |
| [观察者模式](./observer/README.md) | 定义一对多依赖，当对象状态变化时自动通知所有依赖者 | 发布订阅、事件驱动 |
| [状态模式](./state/README.md) | 允许对象在内部状态改变时改变其行为 | 状态机、行为随状态变化 |
| [策略模式](./strategy/README.md) | 定义一系列算法，将其封装并使其可互相替换 | 算法族、运行时切换 |
| [模板方法模式](./template_method/README.md) | 在父类中定义算法骨架，将某些步骤延迟到子类实现 | 骨架方法、钩子 |
| [访问者模式](./visitor/README.md) | 在不修改对象结构的前提下，为对象添加新的操作 | 双分派、操作与结构分离 |

## 如何选择

```
需要处理对象间的行为与通信
    │
    ├─ 请求需要经过多个对象处理，且处理者不固定？
    │       └─ 是 → 责任链模式
    │
    ├─ 需要将操作封装为对象，支持撤销或延迟执行？
    │       └─ 是 → 命令模式
    │
    ├─ 需要解析自定义语言或表达式？
    │       └─ 是 → 解释器模式
    │
    ├─ 需要统一遍历不同集合结构？
    │       └─ 是 → 迭代器模式
    │
    ├─ 多个对象相互依赖，通信复杂？
    │       └─ 是 → 中介者模式
    │
    ├─ 需要保存对象状态以便后续恢复？
    │       └─ 是 → 备忘录模式
    │
    ├─ 一个对象的变化需要通知多个其他对象？
    │       └─ 是 → 观察者模式
    │
    ├─ 对象行为依赖其内部状态，且状态会频繁切换？
    │       └─ 是 → 状态模式
    │
    ├─ 同一行为有多种算法实现，需要运行时切换？
    │       └─ 是 → 策略模式
    │
    ├─ 多个子类共享相同的算法骨架，只有部分步骤不同？
    │       └─ 是 → 模板方法模式
    │
    └─ 需要对一组对象执行多种不相关操作，又不想污染其类？
            └─ 是 → 访问者模式
```
