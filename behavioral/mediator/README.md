# 中介者模式 (Mediator)

## 意图

用中介对象封装一系列对象的交互，使各对象不需要显式地相互引用，从而降低耦合，并可独立地改变它们之间的交互。

## 适用场景

- 组件间存在复杂的网状依赖，修改一个组件会影响大量其他组件（如 UI 控件联动、聊天室系统）
- 希望在不同上下文中复用某些组件，但组件间的相互依赖阻碍了复用
- 需要集中管理多个对象间的交互逻辑，避免逻辑散落在各个组件中
- 组件之间通信的规则需要在运行时动态调整

## 核心结构

```
  ┌─────────────────────────────┐
  │      Mediator (接口)         │
  │  + notify(sender, event)    │
  └─────────────────────────────┘
            ▲
            │ 实现
  ┌─────────────────────────────┐
  │    ConcreteMediator          │
  │  - componentA               │
  │  - componentB               │
  │  + notify(sender, event)    │◄──── 统一处理各组件事件
  └─────────────────────────────┘
       │              │
       ▼              ▼
  ┌──────────┐   ┌──────────┐
  │Component A│   │Component B│
  │- mediator │   │- mediator │
  │+ action() │   │+ action() │
  └──────────┘   └──────────┘

组件之间不直接通信，通过中介者转发所有交互
```

## 优点

- **减少组件间耦合**：组件只依赖中介者接口，互不直接引用
- **集中控制交互逻辑**：所有交互规则聚集在中介者中，便于理解和修改
- **组件可独立复用**：单个组件无需了解其他组件即可独立使用

## 缺点

- **中介者可能变成"神对象"**：随着组件和交互规则增多，中介者逻辑会急剧膨胀，难以维护
- **单点故障**：中介者承载所有交互，一旦出错影响范围广
