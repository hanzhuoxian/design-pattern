# 创建型模式 (Creational Patterns)

创建型模式抽象了实例化过程，将对象的**创建**与**使用**解耦。它们帮助系统在不依赖具体类的情况下完成对象的创建，从而提升灵活性与可复用性。

## 核心思想

- 封装对象创建的细节，客户端只关心接口，不关心具体实现
- 控制何时创建、由谁创建、如何创建
- 在系统演进时，可以替换创建逻辑而不影响使用方

## 模式一览

| 模式 | 意图 | 核心关键词 |
|------|------|-----------|
| [单例模式](./singleton/README.md) | 保证一个类仅有一个实例，并提供全局访问点 | 唯一实例、全局访问 |
| [工厂方法模式](./factory_method/README.md) | 定义创建对象的接口，由子类决定实例化哪个类 | 延迟实例化、子类决定 |
| [抽象工厂模式](./abstract_factory/README.md) | 提供创建一系列相关对象的接口，无需指定具体类 | 产品族、跨平台 |
| [建造者模式](./builder/README.md) | 将复杂对象的构建过程与表示分离 | 分步构建、链式调用 |
| [原型模式](./prototype/README.md) | 通过克隆现有对象来创建新对象 | 克隆、深拷贝 |

## 如何选择

```
需要创建对象
    │
    ├─ 全局唯一实例？
    │       └─ 是 → 单例模式
    │
    ├─ 创建逻辑需要由子类决定？
    │       └─ 是 → 工厂方法模式
    │
    ├─ 需要创建一组相关联的对象（产品族）？
    │       └─ 是 → 抽象工厂模式
    │
    ├─ 对象构建步骤复杂，需要分步骤组装？
    │       └─ 是 → 建造者模式
    │
    └─ 需要复制现有对象，避免重复初始化成本？
            └─ 是 → 原型模式
```
